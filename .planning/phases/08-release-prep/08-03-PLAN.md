  <files>.changeset/pre.json</files>

  <action>
Enter Changesets pre mode to enable beta releases:

1. Enter pre mode with tag "beta":
   ```bash
   pnpm changeset pre enter beta
   ```

This creates .changeset/pre.json with:
```json
{
  "mode": "pre",
  "tag": "beta",
  "initialVersions": {
    "tinyimg-core": "0.0.0",
    "tinyimg-cli": "0.0.0",
    "tinyimg-unplugin": "0.0.0"
  }
}
```

2. Verify pre mode is active:
   ```bash
   cat .changeset/pre.json
   ```

Expected output: mode is "pre", tag is "beta"

3. Test that future changesets will use pre-release versions:
   ```bash
   pnpm changeset
   ```
   (Select "minor" for all three packages, description: "Initial beta release")
   This will create a changeset like .changeset/happy-tigers-cheer.md

4. Exit the interactive changeset creation (we'll create the real changeset in the next task).
  </action>
  <verify>
    <automated>test -f .changeset/pre.json && grep -q '"mode": "pre"' .changeset/pre.json && grep -q '"tag": "beta"' .changeset/pre.json</automated>
  </verify>

<task type="auto">
  <name>Task 4: Create Version PR and update package versions</name>
  <files>packages/*/package.json, CHANGELOG.md</files>
  <read_first>.changeset/*.md</read_first>
  <action>
Run Changesets version command to create Version PR:

1. Run the version command:
   ```bash
   pnpm changeset version
   ```

This will:
- Update package.json files to version 0.1.0-beta.1
- Generate CHANGELOG.md entries
- Delete the consumed changeset file

2. Verify version updates:
   ```bash
   grep '"version"' packages/*/package.json
   ```

Expected: All three packages show "0.1.0-beta.1"

3. Verify CHANGELOG.md generated:
   ```bash
   head -50 CHANGELOG.md
   ```

Expected content:
```markdown
# tinyimg

## 0.1.0-beta.1

### Minor Changes

- 0.1.0-beta.1
- Initial beta release
  - Core library with multi-key management and caching
  - CLI tool for batch image compression
  - unplugin for build-time optimization
  - Support for Vite, Webpack, and Rolldown
```

4. Commit the version changes:
   ```bash
   git add .
   git commit -m "chore: version packages to 0.1.0-beta.1"
   ```

5. Push to GitHub:
   ```bash
   git push origin main
   ```

6. Create a PR or ensure the commit is on main branch (Changesets action will trigger on push to main).

Note: The Changesets GitHub Action should automatically create a Version PR when changesets are present. If not, you may need to manually create a PR with the version changes.
  </action>
  <verify>

  </verify>
  <done>Package versions updated to 0.1.0-beta.1, CHANGELOG.md generated with beta release notes, changes committed</done>
</task>
